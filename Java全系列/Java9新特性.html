    # Java9新特性

    参考文章：

    • https://www.runoob.com/java/java9-new-features.html
    序号 特性
    1 模块系统 (opens new window) 模块是一个包的容器，Java 9 最大的变化之一是引入了模块系统（Jigsaw 项目）。
    2 REPL (JShell) (opens new window) 交互式编程环境。
    3 改进的 Javadoc (opens new window) Javadoc 现在支持在 API 文档中的进行搜索。另外，Javadoc 的输出现在符合兼容 HTML5 标准。
    4 多版本兼容 JAR 包 (opens new window) 多版本兼容 JAR 功能能让你创建仅在特定版本的 Java 环境中运行库程序时选择使用的 class 版本。
    5 集合工厂方法 (opens new window) List，Set 和 Map 接口中，新的静态工厂方法可以创建这些集合的不可变实例。
    6 接口支持私有方法 (opens new window) 支持在接口中使用 private 访问修饰符编写私有方法。
    7 进程 API (opens new window) 改进的 API 来控制和管理操作系统进程。引进 java.lang.ProcessHandle 及其嵌套接口 Info 来让开发者逃离时常因为要获取一个本地进程的 PID 而不得不使用本地代码的窘境。
    8 Stream API (opens new window) 改进的 Stream API 添加了一些便利的方法，使流处理更容易，并使用收集器编写复杂的查询。
    9 优化try-with-resources (opens new window) 如果已经有一个资源是 final 或等效于 final 的变量，您可以在 try-with-resources 语句中直接使用该变量，而无需声明一个新变量。
    10 @Deprecated (opens new window) 注解 @Deprecated 可以标记 Java API 状态，可以表示被标记的 API 将会被移除，或者已经破坏。
    11 内部类的钻石操作符 (opens new window) 匿名类可以使用钻石操作符(Diamond Operator)。
    12 Optional 类 (opens new window) Optional 可以直接转为 Stream，并且新增了stream、ifPresentOrElse、or等方法。
    13 多分辨率图像 API (opens new window) 定义多分辨率图像API，开发者可以很容易的操作和展示不同分辨率的图像了。
    14 CompletableFuture API (opens new window) CompletableFuture 类的异步机制可以在 ProcessHandle.onExit 方法退出时执行操作。
    15 引入HttpClient

    # 集合工厂方法

    AbstractMap抽象类中提供了Entry接口的两种实现，分别是SimpleEntry、SimpleImmutableEntry。前者支持setValue方法改变value值，后者不支持setValue改变value值。

    public class TestUnmodifiableCollection {
    public static void main(String[] args) {
        // 创建不可变List, 不支持增、删、改方法
        List<String> list = List.of("A", "B", "C");
        System.out.println(list);
        // 创建不可变Map
        Map<String, String> map1 = Map.of("A", "Apple", "B", "Boy", "C", "Cat");
        System.out.println("map1:" + map1);
        // 键值对数量超过10个时
        Map<String, String> map2 = Map.ofEntries(
            new AbstractMap.SimpleEntry<>("A", "Apple"),
            new AbstractMap.SimpleEntry<>("B", "Boy"), 
            new AbstractMap.SimpleEntry<>("C", "Cat"));
        System.out.println("map2:" + map2);
        Map<String, String> map3 = Map.ofEntries(
            new AbstractMap.SimpleImmutableEntry<>("A", "Apple"),
            new AbstractMap.SimpleImmutableEntry<>("B", "Boy"), 
            new AbstractMap.SimpleImmutableEntry<>("C", "Cat"));
        System.out.println("map3" + map3);
        // 创建不可变Set, 不支持增、删、改方法
        Set<String> set = Set.of("A", "B", "C");
        System.out.println(set);
    }
}

    不可变集合是指不支持增、删、改等操作的集合。所有继承自AbstractImmutableCollection或AbstractImmutableMap类的集合容器都为不可变集合。

    # ProcessHandle接口

    ProcessHandle接口主要用于增强 java.lang.Process 类。它的实例标识一个本地进程，它允许查询进程状态并管理进程。

    ProcessHandle接口中声明的 onExit() 方法可用于在某个进程终止时触发某些操作。

    ProcessHandle 嵌套接口 Info 来让开发者逃离时常因为要获取一个本地进程的 PID 而不得不使用本地代码的窘境。

    public class TestProcessApi {
    public static void main(String[] args) throws IOException {
        ProcessBuilder pb = new ProcessBuilder("notepad.exe");
        String np = "Not Present";
        Process p = pb.start();
        ProcessHandle.Info info = p.info();
        System.out.printf("Process ID : %s%n", p.pid());
        System.out.printf("Command name : %s%n", info.command().orElse(np));
        System.out.printf("Command line : %s%n", info.commandLine().orElse(np));
        System.out.printf("Start time: %s%n",
            info.startInstant().map(i -> i.atZone(ZoneId.systemDefault()).toLocalDateTime().toString()).orElse(np));
        System.out.printf("Arguments : %s%n", info.arguments().map(a -> String.join(" ", a)).orElse(np));
        System.out.printf("User : %s%n", info.user().orElse(np));
    }
}

    # 改进的 Stream API

    Java9 为 Stream dropWhile、takeWhile、ofNullable方法，并为 iterate 方法新增了一个重载方法。

    public class TestStreamApi {
    public static void main(String[] args) {
        System.out.println("=============takeWhile 1===================");
        Stream<String> s1 = Stream.of("a", "b", "c", "", "e", "f");
        // case1: 有序
        // case2: 当遇到第一个空串时, 断言语句返回false, 停止打印
        s1.takeWhile(e -> !e.isEmpty()).forEach(System.out::println);
        System.out.println("=============takeWhile 2===================");
        // case3: 无序
        // case4: 当遇到第一个使断言语句返回false的元素时，停止打印
        Stream<Integer> s2 = Stream.of(100, 88, 10, 24, 22, 8, null, 99, 101, 1);
        s2.takeWhile(e -> e != null && e % 2 == 0).forEach(System.out::println);
        System.out.println("=============dropWhile 1===================");
        Stream<String> s3 = Stream.of("a", "b", "c", "", "e", "f");
        // case1: 有序
        // case2: 当遇到第一个空串时, 断言语句都会返回true，并开始打印
        s3.dropWhile(e -> !e.isEmpty()).forEach(System.out::println);
        System.out.println("=============dropWhile 2===================");
        // case3: 无序
        // case4: 当遇到第一个使断言语句返回true的元素时，并开始打印
        Stream<Integer> s4 = Stream.of(100, 88, 10, 24, 22, 8, null, 99, 101, 1);
        s4.dropWhile(e -> e != null && e % 2 == 0).forEach(System.out::println);
        System.out.println("=============ofNullable 1===================");
        // ofNullable: 可以预防NullPointerExceptions异常，可以通过检查流来避免null值
        // case1: 返回单个元素流
        System.out.println(Stream.ofNullable("ai tao").count());
        // case2: 返回空流
        System.out.println(Stream.ofNullable(null).count());
        System.out.println("=============iterate 1===================");
        /*
        功能同下面的for循环:
        for (int i = 3; i < 10; i += 3) {
            System.out.println(i);
        }*/
        // public static IntStream iterate(int seed, IntPredicate hasNext, IntUnaryOperator next);
        // tip: e++会导致iterate方法变成无限流(死循环)
        IntStream.iterate(3, e -> e < 10, e -> ++e).forEach(System.out::println);

    }
}

    # 改进的 try-with-resources 语句

    public class TestTryWithResources {
    static String readData1(String message) throws IOException {
        Reader inputString = new StringReader(message);
        BufferedReader br = new BufferedReader(inputString);
        // 改进前
        try (BufferedReader br1 = br) {
            return br1.readLine();
        }
    }

    static String readData2(String message) throws IOException {
        Reader inputString = new StringReader(message);
        BufferedReader br = new BufferedReader(inputString);
        // 改进后
        try (br) {
            return br.readLine();
        }
    }

    public static void main(String[] args) throws IOException {
        System.out.println(readData1("test1"));
        System.out.println(readData2("test2"));
    }
}

    # 改进的 Optional API

    在 Java 9 中，为Optional类新增了stream、ifPresentOrElse、or三个方法。

    **stream方法源码：**将 Optional 转为一个 Stream；如果该 Optional 中包含值就返回包含这个值的 Stream，否则返回一个空的 Stream

    // if optional is non-empty, get the value in stream, otherwise return empty
public Stream<T> stream() {
    if (!isPresent()) {
        return Stream.empty();
    } else {
        return Stream.of(value);
    }
}

    **ifPresentOrElse方法源码：**检测 Optional 中是否存在值；如果该 Optional 中包含值就执行action动作，否则执行emptyAction动作。可以理解为Optional 类的if...else语句作用。

    public void ifPresentOrElse(Consumer<? super T> action, Runnable emptyAction) {
    if (value != null) {
        action.accept(value);
    } else {
        emptyAction.run();
    }
}

    **or方法源码：**检测 Optional 中是否存在值；如果该 Optional 中包含值，返回 Optional 对象本身；否则返回参数中提供的 Optional 中包含的值。

    public Optional<T> or(Supplier<? extends Optional<? extends T>> supplier) {
    Objects.requireNonNull(supplier);
    if (isPresent()) {
        return this;
    } else {
        @SuppressWarnings("unchecked")
        Optional<T> r = (Optional<T>) supplier.get();
        return Objects.requireNonNull(r);
    }
}

    基础使用：

    public class TestOptional {
    public static void main(String[] args) {
        Optional<String> o1 = Optional.of("Ai Tao");
        Optional<String> emptyO2 = Optional.empty();
        // 将 Optional 转换成 Stream
        Stream<String> s1 = o1.stream();
        Stream<String> s2 = emptyO2.stream();
        System.out.println("=============ifPresent===============");
        /*
         * if(Predicate p){
         * Consumer<? super T> action;
         * }
         */
        o1.ifPresent(e -> System.out.println("Value:" + e));
        System.out.println("=============ifPresentOrElse 1===============");
        /*
         * if(Predicate p){
         * Consumer<? super T> action;
         * } else{
         * Runnable emptyAction;
         * }
         */
        o1.ifPresentOrElse(e -> System.out.println("Value:" + e), () -> System.out.println("Empty"));
        System.out.println("=============ifPresentOrElse 2===============");
        emptyO2.ifPresentOrElse(e -> System.out.println("Value:" + e), () -> System.out.println("Empty"));
        System.out.println("=============or 1===============");
        Optional<String> o3 = o1.or(() -> Optional.of("Empty"));
        System.out.println("or:" + o1.get());
        System.out.println("=============or 2===============");
        Optional<String> emptyO4 = emptyO2.or(() -> Optional.of("Empty"));
        System.out.println("or:" + emptyO4.get());
    }
}

    # 改进的 CompletableFuture API

    Java 9 对 CompletableFuture 做了3大改进:

    1. 支持延迟时间（delays）和超时时间（timeouts）；

      public CompletableFuture<T> orTimeout(long timeout, TimeUnit unit);
public CompletableFuture<T> completeOnTimeout(T value, long timeout, TimeUnit unit);
public static Executor delayedExecutor(long delay, TimeUnit unit, Executor executor);
public static Executor delayedExecutor(long delay, TimeUnit unit);

    2. CompletableFuture可以被继承；

      public <U> CompletableFuture<U> newIncompleteFuture();
public Executor defaultExecutor();

    3. 新增工厂方法；

    // 返回一个指定 value 完成的新的CompletionStage，并且只支持 CompletionStage 里的接口。
public static <U> CompletionStage<U> completedStage(U value);
// 返回一个以指定异常完成的 CompletableFuture
public static <U> CompletableFuture<U> failedFuture(Throwable ex);
// 返回一个指定异常完成的新的CompletionStage，并且只支持 CompletionStage 里的接口。
public static <U> CompletionStage<U> failedStage(Throwable ex);
