    # 大数类

    BigInteger与BigDecimal是**java.math包**下的两个类，主要用于操作数据范围超过int或long类型的数据。

    # BigInteger

    使用BigInteger操作大整数。常用方法如下表所示：

    方法 描述
    public BigInteger(String) 将字符串变为BigInteger类型的数据
    public BigInteger add(BigInteger) 加法操作
    public BigInteger substact(BigInteger) 减法操作
    public BigInteger multiply(BigInteger) 乘法操作
    public BigInteger divide(BigInteger) 除法操作
    public BigInteger max(BigInteger) 返回最大值
    public BigInteger min(BigInteger) 返回最小值
    public BigInteger [] divideAndRemainder(BigInteger) 除法操作，返回值的数组第一个元素为除法的商；第二个元素为余数

    # BigDecimal

    使用BigDecimal操作小数的保留位数。常用方法如下表所示：

    方法 描述
    public BigDecimal(double) 将double变为BigInteger类型的数据
    public BigDecimal(int) 将int变为BigDecimal类型的数据
    public BigDecimal(String) 将字符串变为BigDecimal类型的数据
    public BigDecimal add(BigDecimal) 加法操作
    public BigDecimal substact(BigDecimal) 减法操作
    public BigDecimal multiply(BigDecimal) 乘法操作
    public BigDecimal divide(BigDecimal) 除法操作

    # Stack

    Stack的特性就是**“先进后出”**；Stack类是通过继承自Vector类来实现其功能的，而Vector类是继承自AbstractList类并且实现了List接口，所以本质上来说Stack类也是一个List，它具备List中的所有方法。代码如下所示：

    public class Stack<E> extends Vector<E> {}
public class Vector<E>
    extends AbstractList<E>
    implements List<E>, RandomAccess,Cloneable, java.io.Serializable{}
    方法 作用
    boolean empty() 判断栈是否为空。空返回true，非空返回false
    Object peek() 获取栈顶元素，但不移除它。
    Object pop() 获取并移除栈顶元素。
    Object push(Object ele) 入栈
    int search(Object element) 查找指定元素的位置，以 1 为基数。

    应用：

    public class Stack_Demo_01 {
    public static void main(String[] args) {
        //初始化
        Stack<Integer> stack = new Stack<>();
        System.out.println("isEmpty -> " + stack.isEmpty());
        //入栈
        stack.push(5);
        stack.push(4);
        stack.push(3);
        stack.push(2);
        stack.push(1);
        System.out.println("isEmpty -> " + stack.isEmpty());
        System.out.println("pop -> " + stack.pop());
        System.out.println("peek -> " + stack.peek());
        //元素不存在返回-1
        System.out.println("search -> " + stack.search(1));
    }
}

    # Deque

    Deque是一个双端队列接口，继承自Queue接口，Deque的实现类是LinkedList、ArrayDeque、LinkedBlockingDeque，其中LinkedList是最常用的。

    Deque用途：

    //普通队列(一端进另一端出):
Queue queue = new LinkedList();
Deque deque = new LinkedList()
//双端队列(两端都可进出)
Deque deque = new LinkedList()
//堆栈
Deque deque = new LinkedList()

    注意：Java堆栈Stack类已经过时，Java官方推荐使用Deque替代Stack使用。Deque堆栈操作方法：push()、pop()、peek()。

    Deque是一个线性Collection，支持在两端插入和移除元素。

    下表总结了上述 12 种方法：

    Deque接口扩展(继承)了 Queue 接口。在将双端队列用作队列时，将得到 FIFO（先进先出）行为。将元素添加到双端队列的末尾，从双端队列的开头移除元素。从 Queue 接口继承的方法完全等效于 Deque 方法，如下表所示：

    Queue方法 Deque方法
    add(e) addLast(e)
    offer(e) offerLast(e)
    remove() removeFirst()
    poll() pollFirst()
    element() getFirst()
    peek() peekFirst()

    双端队列也可用作 LIFO（先进后出）堆栈。应优先使用此接口而不是遗留 Stack 类。Stack方法完全等效于 Deque 方法，如下表所示：

    Stack方法 Deque方法
    push(e) addFirst(e)
    pop() removeFirst()
    peek() peekFirst()

    # Pair

    Pair是一个键值对类，在 javafx.util 包中。当一个函数返回两个值并且两个值都有重要意义时我们一般会用Map的key和value来表达，但是这样的话就需要两个键值对，用Map映射去做处理时，此时的key相当于value的一个描述或者引用，而具体的信息都保存在value中，我们可以通过key去获取对应的value。但是当key和value都保存具体信息时，我们就需要用到Pair对了。Pair对也是键值对的形式。

    Pair<String, String> pair = new Pair<>("aitao", "male");
System.out.println(pair.getKey());//aitao
System.out.println(pair.getValue());//male

    # LinkedHashMap

    LinkedHashMap是HashMap的一个子类，在HashMap存储结构的基础上，利用了双向链表来保证添加节点和访问节点的顺序性。与HashMap不同之处在于，它内部是通过Entry类来替代HashMap中的Node类的。

    Entry类的结构声明如下图所示：

    LinkedHashMap节点存储方式大概就是这样的了。

    与HashMap不同之处在于，LinkedHashMap能够保证插入节点和访问节点的顺序序，因此它的内部提供了accessOrder属性，可以在实例化时指定该属性值。当acessOrder为false时，即指定的是插入顺序，值为true时即指定的是访问顺序。并且LinkedHashMap中还提供了头指针（head）和尾指针（tail）来记录双向链表的头部节点和尾部节点。

    插入操作

    LinkedHashMap与HashMap插入节点的操作基本一致，不同之处是LinkedHashMap实现了afterNodeInsertion和afterNodeAccess方法

    afterNodeInsertion：移除链表中的最旧的节点值，即删除头节点。在LinkedHashMap中该方式没有啥作用，主要是源码中的removeEldestEntry方法恒返回false

    afterNodeAccess：将指定节点移动到链表尾部。

    访问操作

    与HashMap的get方法不同之处在于LinkedHashMap增加了双向链表的更新逻辑，也就是将被访问过的节点移动到链表尾部。这个逻辑只有accessOrder值为true时才会被执行，具体源码如下：

    总结：

    LinkedHashMap相对于HashMap，增加了双向链表来维护插入节点与访问节点的顺序性，扩展了HashMap中的存储方式，提供了before和after指针，其他处理逻辑与HashMap基本一致，同样存在并发安全问题。

    # Unsafe

    java不能直接访问操作系统底层，而是通过Unsafe类提供了硬件级别的原子操作，主要的功能有：

    分配/释放内存

    //分配内存
public native long allocateMemory(long l);
//扩充内存
public native long reallocateMemory(long l, long l1);
//释放内存
public native void freeMemory(long l);

    定位字段内存位置 / 修改字段值

    字段定位：

    public static final int ARRAY_INT_BASE_OFFSET;
public static final int ARRAY_INT_INDEX_SCALE;

//返回给定field的内存地址偏移量，这个值对于给定的filed是唯一的且是固定不变的
public native long staticFieldOffset(Field field);
//获取对象中offset偏移地址对应的整型field的值,支持volatile load语义
public native int getIntVolatile(Object obj, long l);
//获取对象中offset偏移地址对应的long型field的值
public native long getLong(Object obj, long l);
//获取数组第一个元素的偏移地址
public native int arrayBaseOffset(Class class1);
//获取到数组中元素的增量地址
public native int arrayIndexScale(Class class1);

static {
    ARRAY_INT_BASE_OFFSET = theUnsafe.arrayBaseOffset(int[].class);
    ARRAY_INT_INDEX_SCALE = theUnsafe.arrayIndexScale(int[].class);
}

    数组元素定位：

    Unsafe类中有很多以BASE_OFFSET结尾的常量，这些常量值是通过arrayBaseOffset方法得到的，arrayBaseOffset方法能获取数组第一个元素的偏移地址。Unsafe类中还有很多以INDEX_SCALE结尾的常量，这些常量值是通过arrayIndexScale方法得到的。arrayIndexScale方法能获取到数组中元素的增量地址。将arrayBaseOffset与arrayIndexScale配合使用，可以定位数组中每个元素在内存中的位置。

    挂起和恢复

    将一个线程进行挂起是通过park方法实现的，调用 park后，线程将一直阻塞直到超时或者中断等条件出现。unpark可以终止一个挂起的线程，使其恢复正常。整个并发框架中对线程的挂起操作被封装在 LockSupport类中，LockSupport类中有各种版本pack方法，但最终都调用了Unsafe.park方法。
